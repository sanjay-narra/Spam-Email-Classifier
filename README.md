\# Spam Email Classification using Machine Learning



\## 📌 Project Overview

This project implements a Spam Email Classification System using multiple Machine Learning algorithms to classify emails as Spam or Non-Spam (Ham).

The system evaluates and compares different classifiers based on Accuracy, Precision, and Recall, and visualizes their performance using comparison graphs.



The project is based on the SpamBase dataset and demonstrates how machine learning techniques can be applied to real-world email filtering problems.



---



\## 🎯 Objective

\- Detect and classify incoming emails as Spam or Ham using supervised learning techniques.

\- Compare the performance of multiple ML algorithms.

\- Identify the best performing classifier based on evaluation metrics.

---



\## 📂 Dataset



\- Dataset Name: SpamBase  

\- Source: UCI Machine Learning Repository  

\- Records: 4601 emails  

\- Features: 57 numerical attributes + 1 class label  

\- Split:

&nbsp; - Training: 80%  

&nbsp; - Testing: 20%  



---



\## ⚙️ Machine Learning Algorithms Implemented



1\. K-Nearest Neighbors (KNN)  

2\. Naive Bayes (BernoulliNB)  

3\. Multilayer Perceptron (MLP)  

4\. Support Vector Machine (SVM)  

5\. Decision Tree  

6\. AdaBoost  

7\. Random Forest  

8\. Convolutional Neural Network (CNN)



---



\## 📊 Evaluation Metrics



Each model is evaluated using:

\- Accuracy  

\- Precision  

\- Recall  



Graphs are generated to compare the performance of all algorithms.



---



\## 🖥️ Application Flow



1\. Upload SpamBase Dataset  

2\. Preprocess dataset and split into train/test  

3\. Run KNN, Naive Bayes, and MLP algorithms  

4\. Run SVM, Decision Tree, and AdaBoost algorithms  

5\. Run Random Forest and CNN models  

6\. Generate Accuracy comparison graph  

7\. Generate Recall comparison graph  

8\. Generate Precision comparison graph
